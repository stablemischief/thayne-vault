## AgFin — Major Architectural Issue Identified

Two foundational bugs found during manual testing:

### Bug 1: Document fields vs Module fields not separated
- App has NO formal document-level field namespace layer
- Extraction pipeline writes directly to module fields (M1.*, HIE_*, etc.) at upload time
- Root cause: `apply_schedule_f_hie_mapping()` called inside extraction pipeline (extraction_orchestrator.py lines 175, 189; batch_upload_service.py lines 766, 779, 1117)
- Fix: remove those calls from extraction; create new `populate_module_fields(application_id)` service called ONLY at review trigger time

### Bug 2: Spouse DL overwrites applicant name
- `update_application_name_from_dl()` called for ALL DL uploads regardless of owner type
- Fix: add `document_owner_type != SPOUSE` guard in extraction_orchestrator.py lines 263-266
- Also: spouse DL extracts to same M1.* keys as applicant — need SPOUSE_DL.* → DL.spouse_* prefix

### Correct Document Field Naming Convention (James confirmed)
- ALL fields from a document type use that document's prefix
- Driver's License (both applicant AND spouse): `DL.` prefix
  - `DL.applicant_full_name`, `DL.applicant_dob`, etc.
  - `DL.spouse_full_name`, `DL.spouse_dob`, etc.
- NOT: SPOUSE_DL.* (that's wrong)
- Other prefixes: SF.*, FSA.*, APH.*, BAL.*, LEASE.*, ORG.*, EQ.*, INS.*
