## System Stabilization Session — 2026-04-09 (Claude Code)

Full-day stabilization session with James. All major defects resolved. System now fully operational.

### Issues Resolved

**1. Lock contention / "unable to open database file" (CRITICAL — ROOT CAUSE)**
Removed `CLAUDE_CODE_ENTRYPOINT=cli` from `engine.py`. This env var was forcing every Agent SDK subprocess (one per Slack message) to share `~/.claude/` SQLite state with interactive Claude Code terminal sessions. Concurrent access caused deadlocks. Removing it gives SDK subprocesses isolated auth — no shared state, no contention.

**2. Duplicate responses**
Discord bot and Slack bot were running simultaneously and sharing the session DB. Unloaded the Discord bot via launchctl. James uses Slack exclusively.

**3. OpenClaw gateway unloaded**
Was running as a KeepAlive service, charging ~$0.15–0.20/hr API fees while idle. Unloaded. Can be reactivated if needed for non-Claude models.

**4. SQLite hardening**
Added WAL checkpoint recovery on startup and 3-attempt retry logic with 2s delay in `session.py`'s `_connect()` for any remaining CANTOPEN errors.

**5. Proactive session reset**
Sessions were crashing at 10+ turns / $1.19 when SDK hit memory limits. Added reset thresholds: 8 turns or 50% of budget ($1.00). Fires `memory_flush.py` in background before clearing session.

**6. Session continuity across resets (CRITICAL)**
When a session reset at turn 8, the new session had zero context from the prior conversation. Fixed with a three-part mechanism in `engine.py`:
- In-memory buffer captures last 6 user/assistant exchange pairs per channel
- On reset: buffer serialized to `.claude/data/state/prior-context-slack-{channel_id}.json`
- On new session: state file loaded, formatted as "Prior conversation context" block, injected into context header, file deleted
Turn 9 now sees a summary of turns 1–8. Committed as `feat(engine): add session continuity across proactive resets`.

**7. Image handling in @mentions**
Image download was only wired to DM handler, not @mention handler. Fixed in `slack_bot.py` — both handlers now process attached files identically.

**8. All launchd plists fixed**
Heartbeat, reflection, and archon-check plists had stale `/Users/james/` and `/Volumes/jw-dev/` paths from original machine setup. Fixed all paths to `/Users/jw-dev/`, uv path to `/opt/homebrew/bin/uv`, added USER/LOGNAME/SSL_CERT_FILE env vars. All reloaded and verified. Archon-check changed from weekly (Monday) to daily at 9am.

**9. Vault detached into separate git repo**
`thayne-vault/` was an untracked local directory with no backup. Created private GitHub repo `stablemischief/thayne-vault`. Initialized vault as own git repo, pushed 76 files (all Memory, Projects, inbox content).

**10. Vault git-sync automation**
Installed simonthum/git-sync script and wrapper. Created `com.thayne-brain.vault-sync` launchd service running every 2 minutes. First run immediately detected and pushed today's daily log update. Vault now auto-commits and syncs all changes (heartbeat writes, Obsidian edits, reflection output) to GitHub within 2 minutes.

**11. Gmail OAuth fixed**
`setup_auth.py` was failing: `run_console()` was removed in newer `google-auth-oauthlib`. Fixed to `run_local_server(port=0)`. James authorized via browser with `james@whitfieldjames.com`. Gmail API verified working.
